## Part 1: Fresh Mac Mini Build — Three Options

You have a working Tier 2 RAG system on your current Mac Mini. To deploy it on a new Mac Mini, there are three approaches, each with different tradeoffs.

### Option A: Script-Based Rebuild (Recommended)

Copy your scripts and config to the new machine, run the automated setup. This is the cleanest approach because it creates a fresh environment with no accumulated cruft from the build process.

**Time:** 45–60 minutes (plus 30–60 min for indexing with contextual retrieval)

**Difficulty:** Low — follow the checklist step by step

**Result:** Identical system, clean environment

**The checklist below reflects the Tier 2 system including Ollama, nomic-embed-text, hybrid search, and contextual retrieval.**

**1. Install Homebrew:** /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

**2. Install Python 3.12:** brew install python@3.12

**3. Install Ollama:** brew install ollama &amp;&amp; brew services start ollama

**4. Pull embedding model:** ollama pull nomic-embed-text

**5. Create venv:** /opt/homebrew/bin/python3.12 -m venv ~/claude-memory/venv

**6. Activate:** source ~/claude-memory/venv/bin/activate

**7. Install Python packages:** pip install chromadb anthropic tiktoken python-docx openpyxl python-pptx pdfplumber rank\_bm25 numpy requests

**8. Set API key in ~/.zshrc:** export ANTHROPIC\_API\_KEY='your-key'

**9. Create folder structure:** mkdir -p ~/claude-memory/{scripts,conversations,vectordb,summaries/{session,weekly},config,logs,cache,backups}

**10. Copy scripts from backup:** cp ~/Documents/claude-memory-backup/scripts/* ~/claude-memory/scripts/

**11. Copy config:** cp ~/Documents/claude-memory-backup/config/settings.json ~/claude-memory/config/

**12. Export conversations from Claude.ai, run import + index + summarize:** bash ~/claude-memory/scripts/maintain.sh

**13. Create RAG Inbox:** mkdir -p ~/Library/Mobile\ Documents/com~apple~CloudDocs/RAG\ Inbox/{acme,school,church,construction-management,food,general}

**14. Index documents:** python3 ~/claude-memory/scripts/index\_documents.py

**15. Set up launchd watchers:** Copy .plist files to ~/Library/LaunchAgents/ and load them

**16. Install OpenClaw, configure SOUL.md, BOOT.md, and the RAG skill** b2

**17. Restart gateway and test:** openclaw gateway restart

### Option B: Full System Clone via rsync

Copy the entire ~/claude-memory/ directory and ~/.openclaw/ directory to the new machine. Fastest approach but carries over any artifacts from the build process.

- **Time:** 15–20 minutes
- **Difficulty:** Very low
- **Risk:** Old cached files, stale .pyc, or broken symlinks may carry over

Steps: Install Homebrew, Python 3.12, Ollama, and pull nomic-embed-text on the new machine. Then rsync -av ~/claude-memory/ newmac:~/claude-memory/ and rsync the OpenClaw config. Create the venv on the new machine, install packages, and test. The vectordb transfers intact — no re-indexing needed.

### Option C: Automated Install Script

Create a single bash script that does everything in Option A automatically. This is the best approach if you plan to deploy on multiple machines or want a repeatable, hands-off installation.

- **Time:** 2–3 hours to write and test the script, then 5 minutes per machine
- **Difficulty:** Medium to write, trivial to run
- **Best for:** Multiple deployments, client installations, or selling the system

The script would install all dependencies, create the folder structure, copy scripts from a USB drive or cloud location, configure the API key (prompted), set up launchd watchers, and run the initial index. This is the foundation for the commercialization options below.

## Part 2: Packaging the RAG System for Sale

Your system solves a real problem: giving small businesses and professionals persistent AI memory that runs locally, stays private, and costs almost nothing to operate. There are several ways to commercialize this, from simple to complex.

### Option 1: Done-For-You Service (Easiest, Highest Margin)

Sell the installation as a consulting service. You show up (physically or remotely), set up the Mac Mini, configure everything, train the client on how to drop files into the RAG Inbox and use Telegram. Charge a flat setup fee plus optional monthly support.

| **Component**         | **Details**                                                                    |
|-----------------------|--------------------------------------------------------------------------------|
| What you sell         | Mac Mini setup + RAG system + OpenClaw + Telegram access + training            |
| Pricing model         | $500–$2,000 setup fee + $50–$200/month support                                 |
| Client provides       | Mac Mini, Anthropic API key, Telegram account                                  |
| Your cost per install | 2–4 hours of your time                                                         |
| Recurring revenue     | Monthly support, maintenance, document re-indexing                             |
| Target market         | Small business owners, real estate investors, attorneys, consultants           |
| Pros                  | No software packaging needed. Start selling immediately. High perceived value. |
| Cons                  | Doesn’t scale beyond your personal time. Income tied to hours worked.          |

This is the fastest path to revenue. You already know how to do the install. The MacDailyNews article about Mac Mini AI agents confirms there’s a growing market of professionals buying Mac Minis specifically for local AI setups. You’d be selling the expertise to make it work.

### Option 2: Automated Installer Package (Medium Complexity)

Create a polished install script (or macOS .pkg installer) that a technically-comfortable user can run themselves. Package it with documentation, a quick-start guide, and email support. Sell as a digital download.

| **Component**   | **Details**                                                            |
|-----------------|------------------------------------------------------------------------|
| What you sell   | Installer script + documentation + email support                       |
| Pricing model   | $99–$299 one-time + optional $29/month support tier                    |
| Delivery        | Digital download (Gumroad, Lemon Squeezy, or your own site)            |
| What’s included | All Python scripts, config templates, launchd plists, setup guide      |
| Client provides | Mac Mini, Anthropic API key, basic Terminal comfort                    |
| Pros            | Scales infinitely. Sell while you sleep. Low marginal cost.            |
| Cons            | Requires support for edge cases. Not suitable for non-technical users. |

The installer would be a bash script that checks prerequisites, installs dependencies, creates the folder structure, deploys your scripts, and walks the user through API key setup and first indexing. macOS also supports .pkg installers for more polished distribution. You’d sell it through Gumroad or similar, with Stripe for payments.

### Option 3: Turnkey Mac Mini + Software Bundle (Highest Revenue)

Buy Mac Minis wholesale (or have the client buy one), pre-install everything, and ship a ready-to-use unit. The client plugs it in, connects to WiFi, opens Telegram, and starts asking questions. Zero setup required on their end.

| **Component**     | **Details**                                                                      |
|-------------------|----------------------------------------------------------------------------------|
| What you sell     | Pre-configured Mac Mini + full RAG system + ongoing support                      |
| Pricing model     | $1,500–$3,000 hardware + software bundle, or Mac Mini + $500 software license    |
| Recurring revenue | $50–$200/month managed service                                                   |
| Target market     | Executives, attorneys, medical professionals — people who want it to “just work” |
| Pros              | Highest perceived value. Strongest recurring revenue. True productization.       |
| Cons              | Hardware logistics. Higher upfront cost. Need inventory or drop-ship model.      |

This is the “Jarvis in a box” model that MacDailyNews described. The market is real — people are already buying Mac Minis for this purpose but struggling with the setup. You’d be selling the last mile.

### Option 4: SaaS / Hosted RAG (Most Scalable, Most Complex)

Instead of shipping hardware, host the RAG system in the cloud and sell access as a subscription. Each client gets their own isolated ChromaDB instance, their own document pipeline, and access via a web UI or Telegram.

| **Component**   | **Details**                                                                                      |
|-----------------|--------------------------------------------------------------------------------------------------|
| What you sell   | Hosted RAG memory service with web dashboard                                                     |
| Pricing model   | $29–$99/month subscription based on document count / query volume                                |
| Infrastructure  | VPS or cloud Mac instances (e.g., MacStadium, AWS Mac instances)                                 |
| Pros            | Highest scalability. True passive income. Recurring revenue.                                     |
| Cons            | Loses the privacy-first local advantage. Significant infrastructure work. Ongoing hosting costs. |

This contradicts one of the system’s core selling points (privacy, fully local). But for clients who don’t want to manage hardware, it’s a valid model. You’d need multi-tenant isolation, authentication, and a web interface.

## Commercialization: Key Considerations

### What You Can and Cannot Sell

- **Your scripts:** 100% yours. You wrote them (with Claude’s help). Fully sellable.
- **ChromaDB:** Apache 2.0 license. Free for commercial use.
- **Ollama:** MIT license. Free for commercial use.
- **nomic-embed-text:** Apache 2.0 license. Free for commercial use.
- **OpenClaw:** Check the license. It’s open source but you’d need to verify commercial use terms.
- **Anthropic API:** Each client needs their own API key. You cannot resell API access. Anthropic’s terms require each user to have their own account.
- **Python libraries:** All MIT or Apache licensed. No restrictions.

### Recommended Path

Start with Option 1 (Done-For-You Service) immediately. It requires zero additional development. Take on 3–5 clients, charge $1,000–$1,500 per setup plus $100/month support. Use those engagements to refine the install process and identify what clients struggle with.

In parallel, develop the Option 2 automated installer. Every time you do a manual install, document what you did and automate another step. After 5 installs, your installer script will cover 90% of cases. Then sell the script as a digital product for passive income alongside the service offering.

Option 3 (Turnkey Mac Mini) is the eventual product if demand justifies it. But don’t invest in hardware inventory until you have paying clients proving the market.
